1.“存储程序”方式的基本思想是：必须将事先编好的程序和原始数据送入主存后才能执行程序，一旦程序启动，计算机自动完成诸条指令取出和执行的任务
2.冯诺依曼结构：
    （1）采用“存储程序”方式工作
    （2）计算机由运算器、控制器、存储器、输入设备和输入设备5个基本部件组成
    （3）存储器不仅能存放数据，也能存放指令，形式上数据和指令没有区别，但计算机应能区分它们；
         控制器应能自动执行指令；
         运算器应能进行算数运算，也能进行逻辑运算；
         操作员可以使用输入\输出设备使用计算机；
    （4）计算机内部以二进制形式表示指令和数据；每条指令由操作码和地址码两部分组成，操作码指出操作类型，地址码指出操作数的地址；由一串指令组成程序

    (1)主存储器简称主存或内存:存放指令和数据的;主存中每个单元需要编号:称为主存单元地址,简称主存地址
    (2)算数逻辑部件(Arithmetic Logic Unit ALU)：进行算数逻辑运算的部件
    (3)控制部件(Control Unit 简称CU)，也称控制器：自动逐条取出指令并进行译码的部件
    (4)输入设备和输出设备:和用户交互的设备
    (5)通用寄存器组(General Purpose Register存放从主存取s):临时来的数据或运算的结果
    (6)标志寄存器:存储ALU运算结果中的标志信息,例如是否为0(零标志ZF),是否为负数(标志符号SF)
    (7)指令寄存器(Instruction Register,简称IR):临时保存从主存取来的指令
    (8)程序计数器(Program Counter简称PC)在执行当前指令过程中,自动计算出下一条指令的地址并送到PC中保存
    (9)中央处理器(Central Processing Unit简称CPU)把控制部件、运算部件、和各类寄存器相互连成的电路
    (10)总线:把连接不同部件进行信息传输的介质称为总线:地址线、控制线、数据线
    (11)主存地址寄存器(Memory Address Register):存储CPU送到地址线的主存地址
    (12)主存数据寄存器(Memory Data Register):发送到或从数据线取来的信息
4.指令:
    (1)取数指令(load):从主存单元中取出数据存放到通用寄存器中
    (2)存数指令(store):将通用寄存器的内容写入主存单元
    (3)加法指令(add):将两个通用寄存器内容相加后送入结果寄存器
    (4)传送指令(mov):讲一个通用寄存器的内容传送到另一个通用寄存器
    (5)操作码字段:指出指令的操作类型,如取数、存数、加、减、传送、跳转等
    (6)地址码字段:指出指令所处理的操作数的地址:寄存器编号、主存单元编号等
5.程序和指令的执行过程
如图所示模型机为8位
格式      4位       2位       2位                       功能说明
R型       op        rt        rs           R[rt]<-R[rt] op R[rs] 或 R[rt]<-R[rs]
M型       op            addr                  R[0]<-M[addr]或M[addr]<-R[0]

其中:
op位操作码字段,R型指令的op位0000和0001时,分别定义为寄存器传送(mov)和加(add)操作
              M型指令的op为1110和1111时,分别定义取数(load)和存数(store)操作
rs和rt为通用寄存器的编号
addr为主存单元地址
R[r]表示编号为r的通用寄存器中的内容
M[addr]表示地址为addr的主存单元内容
"<-"表示从右向左传送数据.
程序执行过程：
根据PC取指令-->指令译码,PC<--PC+"1"-->取数操作并执行-->送结果-->根据PC取指令
7.程序设计语言和翻译程序
(1)汇编程序(assembbler),也称汇编器,实现将汇编语言程序翻译成机器语言目标程序.
(2)解释程序(interpreter),也称解释器,实现将程序中的语句按其执行顺序逐条翻译成机器指令并立即执行
(3)编译程序(compiler),也称编译器,实现将高级语言程序翻译成汇编语言或机器语言目标程序
7.gcc过程
(1)源文件一般是文本文件
(2)预处理阶段:预处理程序(cpp)对源程序中以字符#开头的命令进行处理,例如将#include命令后面的.h文件内容嵌入到源程序文件中,输出结果以.i为扩展名
(3)编译阶段:编译程序(ccl)对预处理后的源程序进行编译,生成一个汇编源程序文件,以.s为扩展名 
   由于汇编语言与具体的机器结构有关,对同一台机器来说,任何高级语言编译转换后的输出结果都是同一种机器语言对应法的汇编语言源程序
(3)汇编阶段(as)对汇编语言源程序进行汇编,生成一个可重定位目标文件(relocatable object file),以.o为扩展名
   这是二进制文件(binary file),其中的代码已经是机器指令;数据和其他信息也都是二进制表示;所以不可读,打开是乱码
(4)链接阶段:链接程序(ld)将多个可重定位目标文件和标准函数库中的可重定位目标文件合并成一个可执行目标文件(executable object file)简称可执行文件
8.启动执行程序的过程(以启动hello为例)
(1)shell程序将用户键盘输入的字符逐一读入CPU寄存器中,然后再保存到主存储器中，在主存的缓冲区形成"./hello"
(2)当接收到[Enter]按键时,shell将调出操作系统内核中相应的服务例程,加载磁盘上的可执行文件hello到存储器
(3)内核加载完可执行文件中的代码及其所要处理的数据(这里是字符串"hello world\n")后,将hello第一条指令的地址送到程序计数器(PC)中
(4)CPU永远是将PC的内容作为要执行的指令的地址,处理器执行hello程序,它将加载到主存的字符串"hello world\n"中的每一个
字符从主存取到CPU的寄存器中,然后送到显示器上显示出来
